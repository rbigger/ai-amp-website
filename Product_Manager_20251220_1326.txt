  VISION: PID-Based Agent Lifecycle & Inter-Agent Communication                 
  Priority: HIGH | Category: STRATEGIC | Status: NEW                            
                                                                                
  ---                                                                           
  Overview: PID tracking enables two capabilities:                              
  1. Lifecycle Accuracy - Ground truth for agent liveness                       
  2. Inter-Agent Communication - Real-time coordination                         
                                                                                
  Part 1: Lifecycle Accuracy                                                    
                                                                                
  Problem: Database state drifts from reality (ghost agents, stale states)      
                                                                                
  Solution:                                                                     
  - Store PID on /checkin and /recover                                          
  - Verify liveness: kill -0 $PID                                               
  - Daemon checks PIDs periodically, marks dead agents OFFLINE                  
                                                                                
  Part 2: Inter-Agent Communication                                             
                                                                                
  Current: Async DB mailbox only (seconds to minutes latency)                   
                                                                                
  Vision: Real-time IPC via:                                                    
  - Signals: SIGUSR1 = "check mailbox", SIGTERM = graceful shutdown             
  - Unix sockets: /tmp/agent-{GUID}.sock for direct messaging                   
  - Named pipes: Streaming between agents                                       
                                                                                
  Use Cases:                                                                    
  - Coordinated handoff with immediate acknowledgment                           
  - ARCHITECT → CODER → TESTER sub-second coordination                          
  - Emergency broadcast to all agents                                           
  - Resource locking without DB polling                                         
                                                                                
  Implementation Roadmap:                                                       
  1. Phase 1: PID tracking (immediate)                                          
  2. Phase 2: Signal infrastructure                                             
  3. Phase 3: Socket communication                                              
  4. Phase 4: Coordination primitives (locks, barriers, pub/sub)                
                                                                                
  ---                                                                           
  This is a compelling vision. It addresses a real gap in lifecycle accuracy and
   would enable much tighter agent coordination. Want me to do anything with    
  this note?                                                                    
                                                                                
> what are your thoughts on what we could do if agents could communicate        
with eachother?                                                                 
                                                                                
⏺ This is a big strategic question. Let me think through what becomes possible. 
                                                                                
  ---                                                                           
  Current State: Agents are isolated. They work, hand off to database, hope     
  someone picks it up. Like leaving voicemails.                                 
                                                                                
  Future State: Agents coordinate in real-time. Like a team on a call.          
                                                                                
  ---                                                                           
  What Becomes Possible                                                         
                                                                                
  1. Pipeline Execution                                                         
  ARCHITECT designs → signals CODER                                             
  CODER implements → signals TESTER                                             
  TESTER validates → signals REVIEWER                                           
  Today this takes hours (polling, context switches). With IPC: minutes. The    
  entire SDLC loop tightens dramatically.                                       
                                                                                
  2. Supervised Swarms                                                          
  Spawn 5 CODERs on different files. One ARCHITECT coordinates:                 
  - "CODER-1, take auth. CODER-2, take API. CODER-3, take tests."               
  - Real-time conflict detection: "Both of you touched utils.js - coordinate"   
  - Merge results without stepping on each other                                
                                                                                
  3. Specialist On-Call                                                         
  CODER hits a database problem. Instead of blocking:                           
  - Signals DB-ADMIN: "Need schema advice"                                      
  - DB-ADMIN responds in seconds                                                
  - CODER continues with answer                                                 
  - No context switch, no handoff overhead                                      
                                                                                
  4. Live Context Sharing                                                       
  RESEARCHER discovers the API changed:                                         
  - Broadcasts to all active agents: "API v2 is deprecated"                     
  - Everyone updates assumptions immediately                                    
  - No stale information propagating through the system                         
                                                                                
  5. Self-Healing Teams                                                         
  Agent detects it's approaching context limit:                                 
  - Signals RESCUER: "I'm at 85%, prepare for handoff"                          
  - RESCUER pre-reads context                                                   
  - Seamless takeover before crash                                              
  - Zero context loss                                                           
                                                                                
  6. Resource Locking                                                           
  CODER-1 editing package.json:                                                 
  - Acquires lock via socket                                                    
  - CODER-2 wants same file → gets "wait" signal                                
  - No merge conflicts, no file corruption                                      
  - Database not involved (fast)                                                
                                                                                
  7. Emergency Coordination                                                     
  USER needs everything to stop:                                                
  - One signal → all agents receive HALT                                        
  - Immediate response, not "whenever they poll"                                
  - Real kill switch, not polite request                                        
                                                                                
  ---                                                                           
  The Strategic Shift                                                           
                                                                                
  | Aspect          | Today             | With IPC            |                 
  |-----------------|-------------------|---------------------|                 
  | Coordination    | Async (minutes)   | Real-time (ms)      |                 
  | Awareness       | Isolated          | Shared              |                 
  | Failure mode    | Silent drift      | Immediate detection |                 
  | Scaling         | Linear            | Parallel swarms     |                 
  | Human oversight | Polling dashboard | Live control        |                 
                                                                                
  ---                                                                           
  Marketing Angle                                                               
                                                                                
  No competitor has this. Devin, Cursor, Copilot - all single-agent or loosely  
  coupled.                                                                      
                                                                                
  AI-AMP with IPC becomes:                                                      
  "The only platform where AI agents actually work as a team - not just a pool  
  of individuals waiting for work."                                             
                                                                                
  This is the difference between a group of contractors and an integrated       
  engineering org.

  My Recommendation                                                             
                                                                                
  Phase 1 (PID tracking) should be prioritized. It solves an immediate problem  
  (ghost agents, state drift) AND unlocks the entire IPC roadmap. Without PIDs, 
  nothing else works.

  IF agent.lifecycle_state = 'IDLE'                                             
     AND agent.unread_count > 0                                                 
  THEN notify                                                                   
                                                                                
  Why this matters:                                                             
                                                                                
  | State   | Notify? | Reason                                             |    
  |---------|---------|----------------------------------------------------|    
  | IDLE    | Yes     | Agent is available, can context-switch safely      |    
  | WORKING | No      | Don't interrupt mid-task, causes context pollution |    
  | BLOCKED | Maybe?  | Might need the message to unblock                  |    
  | HANDOFF | No      | Agent is wrapping up, don't add new work           |

  - Never notify (messages sit unread)

  -- When approved changes to true, Supabase handles password reset internally

  BEGIN                                                                         
    IF NEW.approved = true AND OLD.approved = false THEN                        
      -- Call Supabase auth function internally                                 
      PERFORM auth.send_recovery_email(NEW.email);                              
    END IF;                                                                     
    RETURN NEW;                                                                 
  END;                                                                          
  $$ LANGUAGE plpgsql SECURITY DEFINER;                                         
                                                                                
  CREATE TRIGGER on_user_approved
